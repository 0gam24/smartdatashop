# ADR 0011 — 멀티 에이전트 자체 결정 + SEO·AEO·GEO 통합 적용

- 일자: 2026-05-26
- 상태: Accepted
- 관련: ADR 0006 (4기준), ADR 0010 (네트워크 폐기), `docs/PURPOSE.md` v1.1

## 배경

운영자 명시 지시(2026-05-26 KST):
> "모든 선택사항은 내 웹사이트 목적에 맞게 모든 에이전트 소환해서 논의해서 결정해. 앞으로도 계속."
> "모든 포스팅에는 SEO AEO GEO를 반영해서 포스팅해야해."

두 가지 항구 룰을 한 ADR에 묶어 박는다.

## 결정

### D1. 모든 의사결정은 7+ 에이전트 자체 합의

**대상**: 룰·메모리·문서·콘텐츠 작성·펄스 발행·기존 펄스 보강 등 *작은 변경*

**금지**: 운영자에게 "옵션 A/B/C 중 어떻게 진행할까요?" 식 선택 요청. 단, 외부 결제·도메인·env var·계정 신설·사업 모델 근본 변경은 운영자 결정 보존.

**표준 라인업**: SEO / AEO / GEO / 운영자 시간(1.5h/일) / AdSense 리스크 / 편집자·법무 / 데이터 저널 (콘텐츠 결정 시 7명, 페이지·컴포넌트 결정 시 UX·구조 추가)

**합의 형식**: 에이전트별 ✅/⚠/❌ 표 + PURPOSE.md 항목 매핑 근거 + 즉시 실행

**위계**: §1 수익 > §5 절대 마지노선 > §2 운영(1.5h) > §3 콘텐츠(1차 출처) > §4 사이트 운영(단일 독립)

### D2. 매 펄스 SEO·AEO·GEO 통합 체크리스트 의무

세 가지 동시 적용:

- **SEO** (Search Engine Optimization) — 구글·네이버 검색 노출 (기존 적용 중)
- **AEO** (Answer Engine Optimization) — ChatGPT·Perplexity·Bing Copilot 답변에 사용
- **GEO** (Generative Engine Optimization) — LLM이 생성 답변에 인용

체크리스트는 `src/content/CLAUDE.md` `### SEO·AEO·GEO 통합 체크리스트` 섹션에 박힘. 작성 단계에서 자체 검증 후 PR.

### D3. 적용 범위 — 옵션 A 채택 (멀티 에이전트 합의)

신규 펄스에만 자동 적용. 기존 펄스 대량 retrofit은 **금지** (검토 기간 동결 정책 + 1.5h 한도 + fabrication 위험 종합 판단).

검토 통과 후 일괄 retrofit 재검토 가능.

## 합의 회의록 (옵션 A 채택 근거)

| 에이전트 | 옵션 A (룰만) | 옵션 B (룰+8건 retrofit) |
|---|---|---|
| 🔵 SEO | ✅ 안정 | ⚠ URL 변경 위험 |
| 🟢 AEO | ✅ 점진 누적 | ✅ 이상적이나 비용 |
| 🟣 GEO | ✅ 새 글에 강한 신호 | ✅ 더 효과적 |
| 🟠 운영자 시간 | ✅ 5분 | ❌ 4시간 (한도 초과) |
| 🔴 AdSense 리스크 | ✅ 동결 정책 준수 | ❌ 대량 변경 = 영구 reject 위험 |
| ⚖ 편집자 | ⚠ 일관성 우려 | ✅ 일관성 ↑ |
| 📊 데이터 저널 | ✅ 자연 evolution | ⚠ 강제 retrofit |

**결정**: 옵션 A. AdSense 리스크 + 운영자 시간 ❌ 둘 다 옵션 B 거부.

## 결과 (영향)

긍정:
- 운영자 옵션 선택 부담 0 — 1.5h/일 한도 보호
- 일관된 자체 결정 패턴 → 작업 회전 속도 ↑
- 매 펄스가 SEO·AEO·GEO 모두 충족 → AdSense 승인 가속 + 검색·AI 트래픽 가산

부정 / 위험:
- 자체 결정 오판 위험 — 메모리 [[feedback-multi-agent-autonomous]]의 "운영자 결정 보존" 항목 엄수로 회피
- 기존 펄스 일관성 부재 (잠시) — 검토 통과 후 일괄 정리
