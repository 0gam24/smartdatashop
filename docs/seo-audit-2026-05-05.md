## Findings (severity-sorted)

### Critical
None.

### Major
1. Insight detail (`/insight/[slug]/`) emits zero JSON-LD on production.
   `src/layouts/InsightLayout.astro` never accepts/forwards a `jsonld` prop and
   `src/pages/insight/[slug].astro` does not call `buildArticleLD(entry)`.
   Both files are outside the scope guard for this audit, so it is reported
   but not patched. Recommended fix (next ticket): mirror the
   ArticleLayout/[slug] pattern — add `jsonld?: object | object[]` to
   InsightLayout Props, forward to BaseLayout, then in `[slug].astro`
   compute `buildArticleLD(entry)` and a BreadcrumbList for
   `홈 › 인사이트 › {category} › {title}`.
2. Article description on `/2026/05/05/2026-05-05-jongseong-server/` is
   232 characters — exceeds Google's ~160-char SERP truncation. The value
   comes from `entry.data.tldr`, which is reused both as the on-page TLDR
   block (where ~230 chars is fine) and as `<meta name="description">`.
   Recommended fix: introduce optional `metaDescription` frontmatter in the
   pulse content schema and fall back to a 155-char slice of `tldr` when
   absent. Schema/content edits are outside the file allowlist for this
   audit so reported only.
3. `/article` (root-level pulse redirect) is indexable (`200`, present in
   `sitemap-0.xml`). It is a meta-refresh redirect with `<link rel="canonical">`
   pointing to the latest article, so duplicate-content risk is mitigated, but
   News crawlers prefer hard 301/308 redirects. Recommended: replace with a
   Cloudflare Pages `_redirects` rule (`/article  /YYYY/MM/DD/...  302`) or
   exclude `/article/` from the standard sitemap (it is already absent from
   `news-sitemap.xml`). Outside scope for this audit.

### Minor
1. `<title>` tags on most pages are < 60 chars; the article variant
   `5월 종합소득세 신고 첫 주, 홈택스 접속 지연 32% 줄었다 — 스마트데이터샵`
   is 41 Korean chars (~ 64 visual columns). Acceptable but tight; if
   another article ever runs longer, monitor.
2. Home/insight-archive/tag-landing all use the BaseLayout default
   description (157 chars). Functionally fine, but the insight archive
   page deserves its own copy — fixed in this patch (now 73 chars KR).
3. `og:image` is the static `og-default.png` for every URL because no
   pulse currently has a `coverImage`. Once articles begin shipping unique
   covers (already supported by the schema) the dynamic image will flow
   automatically — no code change needed, but Google News favors
   per-article 1200×630 covers, so prioritize cover production.
4. No `<meta name="news_keywords">` on article pages. Deprecated by Google
   News in 2014 but still parsed by some aggregators (e.g., Naver). Optional.
5. CMS edit surface `/admin/` is `noindex,nofollow` and `Disallow`-listed,
   and is correctly absent from both sitemaps — verified.

## Files modified

- `src/lib/jsonld.ts` — added 4 helpers:
  - `buildBreadcrumbLD(items)` → BreadcrumbList
  - `buildItemListLD(items)` → ItemList (descending order)
  - `buildCollectionPageLD(name, description, url)` → CollectionPage
  - `buildSearchActionLD()` → WebSite + SearchAction (sitelinks search box,
    `?q=` URL pattern targeting `/`)

  Existing exports (`buildOrganizationLD`, `buildPersonLD`,
  `buildNewsArticleLD`, `buildArticleLD`, `buildWebPageLD`) untouched.

- `src/layouts/ArticleLayout.astro` — computes BreadcrumbList from
  category + KST date + current URL ( `홈 › 데일리 › {카테고리} › YYYY/MM/DD` ) and
  appends it to whatever `jsonld` the caller passed (single object or
  array), forwarding the merged array to BaseLayout. Article pages now
  emit two JSON-LD blocks: NewsArticle + BreadcrumbList.

- `src/pages/category/[slug]/index.astro` — replaced the prior
  `buildWebPageLD(...)` with three-block stack (CollectionPage + ItemList
  + BreadcrumbList). ItemList contains every pulse (and every insight, if
  any) on the page in descending publication order, matching the on-page
  visual order.

- `src/pages/tag/[group]/[slug]/index.astro` — same three-block pattern;
  BreadcrumbList chain is `홈 › 태그 › {그룹라벨} › {태그값}`.

- `src/pages/insight/index.astro` — was emitting zero JSON-LD; now emits
  CollectionPage + ItemList + BreadcrumbList. Title and description
  promoted to local consts and wired into `<BaseLayout description=...>` so
  the page no longer inherits the site-wide default description.

- `src/pages/index.astro` — added `buildSearchActionLD()` to the existing
  `[Organization, Person]` JSON-LD array so the home now publishes a
  WebSite entity with the sitelinks-search-box `potentialAction`.

## Recommendations not implemented (out of scope)

- **Insight detail JSON-LD** (Major #1) — requires editing
  `src/layouts/InsightLayout.astro` and/or `src/pages/insight/[slug].astro`.
  Both are outside the file allowlist; user should authorize a follow-up
  change.
- **Article description length cap** (Major #2) — requires either a new
  `metaDescription` field on the pulse content schema (`src/content/`,
  blocked) or a shared truncation helper invoked from the route
  (`src/pages/[year]/...` is not in the allowlist either). Easiest fix is
  schema-level.
- **`/article` redirect hardening** (Major #3) — requires either
  Cloudflare `_redirects` (in `public/`, blocked) or an exclude rule in
  `astro.config.mjs` (blocked).
- **Policy-page WebPage LD** (Minor) — would require touching
  `src/layouts/PolicyLayout.astro`, which is not on the allowlist.
- **AMP, `@graph` aggregation, OG image variants per article** — not
  required for News Publisher Center; defer.
- **News sitemap rolling 48-hour purge** — current implementation in
  `src/pages/news-sitemap.xml.ts` already publishes only recent pulses;
  long-term confirm the route filters anything older than 48 h to comply
  with the Google News spec.
