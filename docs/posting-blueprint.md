## 0. 왜 다시 기획했나 (2026-06-15 운영자 지시)

기존 `daily-posting-policy.md`의 키워드 발굴은 **검색어 도구**(Google 자동완성·네이버 키워드도구·GSC)에만 의존했다. 문제: 검색어 도구는 *단어*를 주지 *니즈(사람이 진짜 풀고 싶은 문제)* 를 주지 않는다. 그 결과 글이 "정부가 X 발표" 요약으로 흐르기 쉽다 — AdSense "가치 부족" reject 의 직접 원인.

**보강**: 한국인이 실제 질문을 쏟아내는 **네이버 지식인(Knowledge-iN)** 을 1순위 발굴 소스로 추가한다. 지식인 질문은 그 자체가 ① 롱테일 검색어 ② 독자의 진짜 니즈 ③ FAQ·AEO 즉답 구조 — 세 개를 동시에 준다. 신생·저권위 사이트가 이길 수 있는 **"답이 비어 있는 질문"** 의 보고다.

## 1. 7+ 에이전트 재합의 — SEO/AEO/GEO 재기획

| 에이전트 | 판정 | 재기획 핵심 |
|---|---|---|
| 🔵 SEO | ✅ | 지식인 질문 = 검증된 롱테일 검색어. 질문 그대로 H2/제목에 박으면 검색 의도 100% 정합. 클러스터 내부링크로 권위 결집 |
| 🟢 AEO | ✅ | 질문→즉답 구조가 곧 featured snippet·ChatGPT 인용 형식. 첫 문단 5W1H 즉답 + FAQPage LD 자동 트리거 |
| 🟣 GEO | ✅ | "사람이 묻는 질문에 1차 출처로 답한 글" = LLM 이 가장 인용하기 좋은 형태. 모든 수치 `[^N]` 페어링 유지 |
| 🟠 운영자 시간 | ✅ | 지식인 검색 5분이 후보·니즈·FAQ 골격을 동시에 산출 → 글당 기획 시간 단축. 1.5h/일 한도 내 |
| 🔴 AdSense | ✅ | "요약"이 아니라 "독자 질문 해결" = 원본 가치 신호. 재승인 국면 핵심. 단 무수요·클러스터 밖 질문 양산 금지 |
| ⚖ 편집·법무 | ✅ | YMYL 질문일수록 1차 출처·발표일·단정 완화 엄격. 지식인 *답변*은 출처 아님 — 질문(니즈)만 차용, 사실은 `.go.kr`로 검증 |
| 📊 데이터 저널 | ✅ | 질문에 "얼마/언제/조건" 이 많음 → 표·계산·시계열로 답하면 시각화 가치까지 결합 |

**합의 결론**: 발굴 소스에 지식인을 1순위로 추가하고, 글 구조를 **"독자 질문 해결형"** 으로 표준화한다. 즉시 적용.

## 2. 키워드·니즈 발굴 3소스 (지식인 1순위 추가)

매 글 발행 전 자동 수행. ⚖ 법무 주의: **지식인은 *질문(니즈)* 만 차용한다. 답변 내용·수치는 절대 사실 출처로 쓰지 않는다** — 사실은 반드시 `.go.kr`/`.or.kr` 1차 출처로 교차검증(ADR 0006).

### 소스 A — 네이버 지식인 최다 질문 ★1순위 (니즈 원천)
1. 네이버 지식인에서 클러스터 토픽 검색 (예: `아동수당`, `근로장려금`, `전기요금 복지할인`).
2. **반복되는 질문 패턴**을 5~10개 수집. 답변 많은 질문 = 니즈 강한 질문.
3. 질문을 니즈 유형으로 분류:
   - **자격형** — "나도 받을 수 있나요?" (대상·소득·연령 요건)
   - **금액형** — "얼마 받아요?" (지급액·계산·상한)
   - **시점형** — "언제 신청·지급돼요?" (마감·지급일·소급)
   - **방법형** — "어디서 어떻게 신청해요?" (절차·서류·온라인)
   - **충돌형** — "A 받으면 B 못 받나요?" (중복·감액·우선순위)
4. 가장 많이·간절하게 묻는 질문 = 글의 **제목**과 **H2 뼈대**가 된다.

### 소스 B — 실제 검색어 (winnability 확인)
1. 지식인에서 뽑은 질문을 Google 자동완성·네이버 키워드도구에 넣어 **실검 여부·검색량** 확인.
2. 그 쿼리를 실제 검색해 1페이지 **경쟁 점검** — 대형 매체·정부 독점이면 회피, 블로그·중소사 섞이면 채택(`daily-posting-policy.md` §2-1).
3. GSC 노출 쿼리에 이미 뜬 키워드면 우선순위 ↑ (구글이 인정한 winnable).

### 소스 C — 1차 출처 검증 (발행 게이트)
1. 질문에 답할 사실(금액·요건·일자)을 `.go.kr`/`.or.kr`에서 확인. 검증 불가 → **그 글 탈락**(ADR 0006 #4).
2. deep link 또는 기관 루트 + `accessedAt` 명시.

> **3소스 교집합이 곧 winnable 글**: A(니즈 있음) ∩ B(검색되고 이길 수 있음) ∩ C(1차 출처로 검증됨).

## 3. 니즈 → 구조 대입표 (핵심)

지식인에서 수집한 질문 유형을 글의 섹션·LD에 1:1로 꽂는다. **이 표가 청사진의 심장이다.**

| 지식인 질문 유형 | → 글 섹션 (H2) | → SEO/AEO/GEO 효과 |
|---|---|---|
| 자격형 "나도 받나요?" | ## 누가 받나 — 대상·요건 | 검색의도 정합 + 자격 표(매트릭스) |
| 금액형 "얼마예요?" | ## 얼마를 — 금액·계산 | 표 + 편집자 계산 → GEO 인용 |
| 시점형 "언제예요?" | ## 언제 — 신청·지급 일정 | 시의성 → Discover + 발표일 명시 |
| 방법형 "어떻게 신청?" | ## 신청 방법 — 절차 | 번호 체크리스트 → HowTo LD 자동 |
| 충돌형 "중복 되나요?" | ## A와 B 중복·차이 | 비교 표 + 내부링크 → 클러스터 권위 |
| (니즈 배경) "왜 바뀜?" | ## 배경 — 왜 이렇게 | 원본 가치(분석) → AdSense 신뢰 |
| 잔여 질문 3개+ | ## 자주 묻는 질문 | FAQPage LD 자동 + AEO 즉답 |

**규칙**: 수집한 질문 중 *가장 강한 1개*는 제목으로, *상위 3~5개*는 H2로, *나머지*는 FAQ Q&A로. 질문은 **그대로의 자연어**로 쓴다(검색어와 일치).

## 5. 발행 전 자가검증 (압축 — 풀 체크리스트는 `src/content/CLAUDE.md`)

발행 직전 5초 점검. 하나라도 ❌면 발행 보류.

- ☐ **니즈 출처**: 지식인에서 실제 질문 5+개 수집했나? 제목=최다 질문, H2=상위 질문인가?
- ☐ **winnability**: 그 쿼리 1페이지에 블로그·중소사 섞여 있나(이길 수 있나)?
- ☐ **클러스터**: 🅰 정부 지원금·복지 / 🅱 생활경제 안에 드나? 밖이면 탈락.
- ☐ **1차 출처(ADR 0006)**: 모든 수치·일자에 `[^N]` 페어링 + `.go.kr`/`.or.kr` 검증 + 발표일 본문 명시?
- ☐ **깊이**: 공백 제외 3,000자+ / H2 6+ / 표 1+ / 표 직후 해설 / 배경 섹션 / FAQ 3+?
- ☐ **문체**: 완전 문장(요약체 금지) / 표 셀 ≤40자·화살표 0?
- ☐ **날짜**: PowerShell `Get-Date`로 실제 KST 확인 후 `publishedAt`·발표일 작성?
- ☐ **금지**: 자매/네트워크 링크 0 / 검수 미완 토큰 0 / 미검증 수치 0?
