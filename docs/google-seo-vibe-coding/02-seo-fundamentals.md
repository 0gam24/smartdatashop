# SEO 기본 가이드 (SEO Starter Guide)

> 검색엔진 최적화의 기본기를 정리한 가이드
> 출처: https://developers.google.com/search/docs/fundamentals/seo-starter-guide?hl=ko

## Google 검색은 어떻게 동작하는가
Google은 완전 자동화된 검색엔진으로, **크롤러(Googlebot)** 가 끊임없이 웹을 탐색하여 색인할 페이지를 찾는다.
1. **크롤링(Crawling)** — 페이지 발견 및 다운로드
2. **색인 생성(Indexing)** — 콘텐츠 분석 및 저장
3. **게재(Serving)** — 사용자 검색어에 가장 관련성 높은 결과 제공

> 변경사항 반영에는 몇 시간 ~ 몇 개월이 걸릴 수 있으며, 일반적으로 결과 확인까지 수 주를 권장한다.

---

## Google이 콘텐츠를 찾도록 돕기
- `site:내도메인.com` 검색으로 색인 여부 먼저 확인
- 새 페이지는 주로 **다른 페이지의 링크**를 통해 발견됨
- 큰 사이트는 **사이트맵(sitemap.xml)** 제출 권장
- CSS/JS 등 렌더링 리소스를 Googlebot이 접근 가능해야 함
- Search Console의 **URL 검사 도구**로 Google이 보는 페이지 확인

## 검색 노출에서 페이지 제외하기
- `noindex` 메타 태그
- robots.txt로 크롤링 차단
- 인증/비밀번호 보호

---

## 사이트 구조

### 설명적인 URL 사용
✅ `https://example.com/pets/cats.html`
❌ `https://example.com/2/6772756D707920636174`

### 디렉터리로 주제 그룹화
- `/policies/` (저변경 빈도) vs `/promotions/` (고변경 빈도)
- Google이 각 디렉터리의 변경 주기를 학습 → 크롤링 효율 ↑

### 중복 콘텐츠 처리
- 같은 콘텐츠가 여러 URL로 노출되지 않도록
- 301 리디렉션 또는 `<link rel="canonical">` 사용
- 중복 자체로 페널티는 없지만 크롤링 예산을 낭비함

---

## 콘텐츠 작성 원칙
- **읽기 쉽고 체계적**: 단락, 헤딩, 맞춤법
- **고유성**: 다른 사이트 복사 금지
- **최신성**: 주기적 업데이트, 오래된 페이지 정리
- **신뢰성**: 사람 중심, 경험·전문성 기반
- 독자가 사용할 다양한 검색어를 예상 (예: "샤퀴테리" vs "가공육")
- 산만한 광고·인터스티셜 지양

---

## 링크
- 명확한 **앵커 텍스트** 작성 ("여기 클릭" ❌, "초콜릿 칩 쿠키 레시피" ✅)
- 신뢰 가능한 외부 리소스에 링크
- 신뢰할 수 없는 링크에는 `rel="nofollow"`, `ugc`, `sponsored` 추가
- 사용자 생성 콘텐츠(UGC)의 외부 링크에 자동으로 nofollow 적용

---

## 검색결과 노출 최적화

### 제목 링크 (`<title>`)
- 페이지마다 고유하고, 명확하고, 간결하게
- 회사명·위치·핵심 서비스 포함 가능

### 스니펫 (`<meta name="description">`)
- 페이지에 대한 1~2 문장 요약
- 페이지마다 고유한 메타 디스크립션 작성

---

## 이미지 최적화
- 고화질 이미지 사용
- 관련 텍스트 가까이에 배치
- `<img alt="설명적인 대체 텍스트">` 필수
- 의미 있는 파일명 사용

## 동영상 최적화
- 동영상마다 독립된 페이지에 임베드
- 제목·설명을 설명적으로 작성

---

## 무시해도 되는 SEO 오해
- **메타 키워드(`<meta name="keywords">`)** — Google이 사용하지 않음
- **키워드 반복** — 스팸 정책 위반
- **도메인 키워드** — 순위에 큰 영향 없음
- **콘텐츠 최소/최대 길이** — 정해진 단어 수 기준 없음
- **하위 도메인 vs 하위 디렉터리** — 관리 편의성으로 선택
- **PageRank 단독 신호** — 많은 신호 중 하나일 뿐
- **중복 콘텐츠 페널티** — 직접 조치 없음 (효율 문제만)
- **헤딩 개수/순서 강박** — Google은 의미 순서에 의존하지 않음
- **E-E-A-T를 단일 순위 요소로 봄** — ❌ 아님 (개념적 가이드일 뿐)

---

## 바이브코딩 체크리스트
- [ ] URL이 사람이 읽을 수 있는가
- [ ] `<title>`이 페이지마다 고유한가
- [ ] `meta description`이 50~160자 사이로 작성되었는가
- [ ] 헤딩(`h1`~`h6`)이 의미적으로 구성되었는가
- [ ] 모든 이미지에 `alt` 속성이 있는가
- [ ] canonical URL이 명시되었는가
- [ ] 외부 링크에 적절한 `rel` 속성이 있는가
