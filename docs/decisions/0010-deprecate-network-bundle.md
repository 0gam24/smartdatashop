# ADR 0010 — 네트워크 컬렉트 폐기, 단일 사이트 독립 운영

- 일자: 2026-05-24
- 상태: Accepted
- 관련: **supersedes ADR 0008 (network revenue funnel)** / ADR 0003 (toll-gate matrix, 사실상 무효)

## 배경

2026-05-07 ADR 0008 로 본 사이트는 "메인 hub + 자매 4~9" 네트워크 컬렉트의 핵심으로 정의됐다. 양방향 funnel·합산 트래픽·자매 RSS fetch·톨게이트 등이 의무화됐다.

2026-05-24 운영자 검토 결과:

- 본 사이트 페이지에서 자매 링크를 타고 들어가도 **자세한 내용이 부재**. funnel 끝이 빈약하여 사용자 이탈.
- "한 덩어리" 컨셉이 콘텐츠 작업 시 *상시 인지 부담*을 만들지만, 실제로는 자매에 의존하는 페이지 경험이 작동하지 않음.
- 자매 사이트는 각자 독립 운영하는 편이 운영자 1.5h/일 한도와 일관됨.

## 결정

### D1. 네트워크 컬렉트 컨셉 폐기

본 사이트는 **단일 독립 사이트**로 운영. 합산 트래픽 KPI 폐기.

KPI 재정의: **본 사이트 단독 페이지뷰·AdSense 수익**.

### D2. 자매 사이트는 별도 사업체로 분리

- 자매 4개 (`calculatorhost.com` / `awoo.or.kr` / `asiatop.co.kr` / `iknowhowinfo.com`) 는 각자 독립 운영.
- 본 사이트 하네스(`CLAUDE.md` / 에이전트 / ADR) 가 자매에 영향 미치지 않음. 역도 마찬가지.
- 운영자가 *필요할 때* 자매 사이트로 별도 의뢰. 본 사이트 작업 중 자매 컨텍스트 자동 호출 금지.

### D3. 자매 cross-link 의무 폐기

- 펄스 본문 톨게이트 박스, 페르소나 매핑, "더 깊이 →" 박스 — **신규 작성 금지**.
- 메인 홈 "네트워크 최신" 섹션, 자매 RSS fetch (`scripts/fetch-sister-mirrors.mjs` 등), `sister-mirrors/` 데이터 — *작동은 유지되나 신규 의존 금지*. 점진 제거 후속.
- 페르소나 매핑(`data/sister-mapping.json` 등)도 신규 작성 금지.

### D4. 헌법 문서 아카이브

다음 3 문서는 `docs/_archived/` 로 이동 — 호기심 참조용만, 현행 하네스 영향 0:

- `docs/PROJECT_DEFINITION.md` (5~9 사이트 전체 그림)
- `docs/NETWORK.md` (5~9 사이트 공통 헌법)
- `docs/CATEGORY_MAP.md` (5 사이트 카테고리 매핑)

`docs/PURPOSE.md` 는 v1.1 로 bump (§4 네트워크 목적 삭제).
`CLAUDE.md` 의 "프로젝트 목적 (확장)" / "네트워크 헌법" 섹션 삭제·재작성.

### D5. 코드는 점진 정리

- 자매 mirror fetch / 톨게이트 / 네트워크 인덱스 코드는 **즉시 삭제하지 않음** (CI 영향 위험).
- 후속 PR 로 단계적 제거 — 본 ADR 직후 운영자가 우선순위 결정.
- 신규 작성·신규 의존은 본 ADR 시점부터 즉시 금지.

## 결과 (영향)

긍정:
- 콘텐츠 작업 시 인지 부담 감소 (자매 매핑·페르소나 forwarding 고려 0)
- AdSense 수익 책임 본 사이트로 명확화 — 디버깅·실험 회전 빨라짐
- ADR 0006 4기준이 본 사이트 단독 적용 (이전: "네트워크 영구 차단 위험" 표현 → 단순화)

부정 / 위험:
- 합산 트래픽 multiplier 포기 — 트래픽 KPI 단기 위축 가능
- 자매 backlink SEO 효과 상실 — 도메인 권위 자체 증식 의존
- 기존 자매 mirror 코드 일정 기간 dead code 잔존
