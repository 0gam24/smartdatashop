# 운영자 검수 큐 — 1차 출처 대조 작업표

> **이 문서는 운영자(김준혁)가 직접 1차 출처를 확인하고 `[검수 후 입력]` 자리에 채워 넣는 작업표입니다.**
>
> AI는 정부 통계·기관 발표 일자를 직접 검증할 수 없으며, 이 사이트의 `verifiedBy` 프로토콜은 그 한계를 구조적으로 강제합니다. AI가 "확인했다"고 채워 넣으면 사이트의 신뢰 자산이 무너지므로, 본 작업은 **반드시 운영자가 1차 출처 문서를 펼친 상태에서 직접 수행**해야 합니다.

---

## 작업 절차 (글당 반복)

1. 본 문서에서 글 한 편 선택
2. **frontmatter `sources` URL** 클릭 → 기관 사이트로 이동
3. 사이트 검색·메뉴에서 **구체적인 보도자료/통계 문서**(deep link) 찾기
4. 각 `[검수 후 입력]` 자리의 컨텍스트와 1차 출처 수치를 **한 줄씩 대조**
5. 일치하면 `[검수 후 입력]`을 실제 값으로 교체
6. 각주 마커(`[^N]`)가 없는 곳은 추가 — 본문 끝 footnote definition도 함께 갱신
7. **frontmatter `sources`에 deep link URL 추가** (현재는 기관 루트만 있음)
8. 모든 placeholder가 채워지면:
   - `previewMode: false`로 변경
   - `verifiedBy: ['junhyuk-kim']`로 변경
   - `updatedAt` 추가 (수정 시각)
9. 빌드 확인 후 commit + push

> 💡 **하나라도 1차 출처에서 확인 불가능한 수치는 본문에서 제거하거나 "운영자 직접 추정" 명시.** 미검증 채로 두느니 빼는 편이 신뢰 자산.

## AI가 도울 수 있는 범위

운영자 작업을 줄이기 위해 AI가 보조할 수 있는 영역:

1. ✅ **링크 헬스 체크** — frontmatter sources URL이 200으로 응답하는지 자동 확인 (별도 스크립트 가능)
2. ✅ **Deep link URL 후보 제안** — 기관 사이트 검색 결과를 fetch해 후보 문서 URL 5-10개 제시 (운영자가 그 중 정확한 것 선택)
3. ✅ **Placeholder 매칭** — `[검수 후 입력]` 자리에 운영자가 입력한 값이 frontmatter source의 deep link 텍스트와 일관되는지 검증
4. ✅ **Footnote 정합성** — `[^N]` 마커와 footnote definition이 1:1 대응하는지 검증

**AI가 절대 하지 않는 작업**:
- ❌ Placeholder 자리에 직접 값 채우기
- ❌ 1차 출처 문서를 "확인했다"고 주장하기
- ❌ verifiedBy 자리에 운영자 ID 적기
- ❌ previewMode를 false로 변경하기

이 경계가 사이트의 신뢰 자산을 지키는 마지막 방어선입니다.
